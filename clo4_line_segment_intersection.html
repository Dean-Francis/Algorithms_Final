Educational Overview Sweep line, cross-product tests, and complexity analysis
›

The sweep line algorithm detects all intersections among n line segments more efficiently than brute force. A vertical line sweeps left to right, processing events: left endpoints (insert), right endpoints (remove), and intersections (swap).

Complexity
Brute force: O(n²) — test every pair.
Shamos-Hoey (detect if any): O(n log n).
Bentley-Ottmann (report all k): O((n + k) log n).
For sparse inputs (k ≪ n²) this gives a dramatic speedup.
Cross-Product Orientation Test
Given points p, q, r: compute val = (q−p) × (r−q).
val > 0 → Clockwise (right turn).
val < 0 → Counter-Clockwise (left turn).
val = 0 → Collinear.

Two segments s1=(A,B) and s2=(C,D) intersect if and only if A and B are on opposite sides of line CD, and C and D are on opposite sides of line AB. This is tested in O(1) per pair.
Active Set (BST)
The active set T stores currently-crossing segments ordered by their y-coordinate at the sweep line's current x. When a segment is inserted or removed, only its immediate neighbors in T need to be tested — making each event O(log n). A balanced BST (e.g. red-black tree) supports insert, delete, and predecessor/successor in O(log n).

s1: (0.1, 0.3) → (0.9, 0.7)   s2: (0.2, 0.7) → (0.8, 0.2)

Event 1 — LEFT of s1 (x=0.1): insert s1 into T. No neighbors.
Event 2 — LEFT of s2 (x=0.2): insert s2. Test s1 ∩ s2.
