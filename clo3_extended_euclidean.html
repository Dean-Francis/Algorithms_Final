The Euclidean Algorithm (≈300 BC) computes gcd(a, b) by repeatedly replacing (a, b) with (b, a mod b) until the remainder is zero. It runs in O(log min(a, b)) divisions.

Bézout's Identity
For any integers a and b there exist x, y such that ax + by = gcd(a, b). The extended algorithm finds x and y via back-substitution.
Modular Inverse
If gcd(a, n) = 1 then a⁻¹ mod n = x mod n where ax + ny = 1. This is the foundation of RSA and Diffie-Hellman.
Complexity & Worst Case (Lamé's Theorem)
Time: O(log min(a, b)) divisions — logarithmic in the smaller input.
Worst case: consecutive Fibonacci numbers — the number of divisions never exceeds 5 × log₁₀(min(a, b)).
Try it: the ⚠ Fibonacci preset uses a = 144, n = 89.
Worked Example — gcd(35, 15)
35 = 2·15 + 5  →  15 = 3·5 + 0  →  gcd = 5
Back-sub: 5 = 35 − 2·15  ∴  x = 1, y = −2
Verify: 35(1) + 15(−2) = 5 ✓
RSA Application
RSA generates keys using: gcd(e, φ(n)) = 1 and d = e⁻¹ mod φ(n).
Example: e=3, φ(n)=26 → extGCD(3, 26) gives x=9 → d=9.
Check: 3 × 9 = 27 ≡ 1 (mod 26) ✓ — d=9 is the private decryption exponent.
