Educational Overview Work, span, parallelism analysis and worked examples
›

Parallel Merge Sort extends classic merge sort using a fork-join model. Each recursive call spawns two subtasks that run concurrently, then syncs before merging. On a machine with unlimited processors it achieves polylogarithmic span.

Work vs Span
Work T₁ = O(n log n) — total operations as if sequential.
Span T∞ = O(log² n) — critical path length with infinite processors.
Parallelism
Parallelism = Work / Span = O(n log n / log² n) = O(n / log n). We can usefully employ up to O(n / log n) processors before returns diminish.
Brent's Theorem
With p processors: T_p ≤ W/p + D
Where W = total work, D = span (depth), p = processor count.
Example: n=8, W≈24, D≈9, p=4 → T₄ ≤ 24/4 + 9 = 15 time units.
Parallel Merge — Binary Search Ranking
To merge two sorted halves, each element binary-searches its rank in the other half in O(log n) time. With O(n) processors working simultaneously the entire merge costs O(log n) parallel time — reducing total span to O(log² n).

Example — merging [1,3] and [2,4]:
Element 1: binary-search in [2,4] → rank 0 → output[0]
Element 3: binary-search in [2,4] → rank 1 → output[2]
Element 2: binary-search in [1,3] → rank 1 → output[1]
Element 4: binary-search in [1,3] → rank 2 → output[3]
All four searches run simultaneously → result [1,2,3,4] in O(log 2) = O(1).
Worked Example — Sorting [3, 1, 4, 2]
Split phase (parallel branches execute simultaneously):
[3,1,4,2] → [3,1] ‖ [4,2] → [3] ‖ [1] ‖ [4] ‖ [2]

Base cases (t=0, each single element is trivially sorted):
[3] [1] [4] [2]

Merge level 1 (t=1, two parallel merges):
Merge [3]+[1] → [1,3]    Merge [4]+[2] → [2,4]   simultaneous

Merge level 2 (t=2, final merge):
Merge [1,3]+[2,4] → [1,2,3,4] (4 binary searches in parallel)

Total parallel time: 2 units vs sequential O(n log n) ≈ 8 ops.
With p=2: T₂ ≤ W/p + D = 4 + 2 = 6 units (Brent's bound).
