Greedy Set Cover

Guided walkthrough — concepts, theory, and examples
Page 1 of 4
Let U be some ground set (universe)
U holds every element we need to cover — nothing is left out.
let S be a family of subsets of U, where ∪S = U
Each set Sᵢ covers some elements. Together they span all of U.
Goal: cover all elements using as few sets as possible
Picking every set always works — but that's wasteful. We want the smallest sub-collection C ⊆ S such that ∪C = U.
Finding the smallest number of sets is computationally difficult
With m sets, there are 2ᵐ possible sub-collections to check.
For m = 50 that is over a quadrillion combinations.
Set Cover is NP-hard
No known algorithm solves it exactly in polynomial time.
Instead of searching for the perfect answer, we use a greedy approximation that runs fast and gets close.
Greedy idea: repeatedly pick the set that covers the most uncovered elements
At each round, look at what is still uncovered and choose the set that eliminates the most of it.
Continue until all elements are covered
Repeat the same greedy pick with the remaining uncovered elements until none are left.
Each step reduces uncovered elements as much as possible
This local-best choice is the greedy heuristic. It will not always find the smallest cover, but it gets close — fast.
Start with C ← ∅ and U' ← U
C tracks chosen sets. U' tracks what remains uncovered.
Each round, pick S* = argmax |Sᵢ ∩ U'|
Among all sets, greedily choose whichever covers the most uncovered elements.
Add S* to C, remove from U'. Repeat until U' = ∅
Each iteration covers at least one new element, so the algorithm always terminates.
Greedy Set Cover
Input U, S = {S₁ … Sₘ}
Output C ⊆ S, ∪C = U
C ← ∅
U' ← U
while U' ≠ ∅ :
S* ← argmax |Sᵢ ∩ U'|
C ← C ∪ { S* }
U' ← U' \ S*
return C
Round 1
U' = {1, 2, 3, 4, 5, 6}
|S₁ ∩ U'| = 3 (max) → pick S₁
C = {S₁} uncovered: 3 left
Round 2
U' = {4, 5, 6}
|S₂ ∩ U'| = 2 (max) → pick S₂
C = {S₁, S₂} uncovered: 1 left
Round 3
U' = {6}
|S₃ ∩ U'| = 1 → pick S₃
C = {S₁, S₂, S₃} done ✓
Greedy does not always give the minimum number of sets
Because each pick is locally optimal, some choices can block a globally better solution. Greedy may use more sets than strictly needed.
But it is never far from optimal
The gap is bounded by the harmonic number H(k). Greedy is provably close — this is the approximation guarantee.
The harmonic number H(k)
H(k) = 1 + 1/2 + 1/3 + … + 1/k
H(1) = 1.000
H(2) = 1.500
H(3) = 1.833
H(5) = 2.283
H(10) = 2.929
H(50) = 4.499
Greedy is an H(k)-approximation
|Greedy Cover| ≤ H(k) · |OPT|
where k = size of the largest set in S
Greedy is simple but near-optimal in practice
The algorithm runs in polynomial time — just scan all sets each round and pick the best. Time complexity: O(n · m) where n = |U|, m = |S|.
Performance is logarithmically bounded
H(k) ≈ ln(k) + 0.5772…
For k = 100 sets, H(100) ≈ 5.19 — greedy is at most ~5× from optimal. Often much better in practice.
Set Cover is NP-hard
No polynomial-time algorithm is known to find the minimum cover exactly. With m sets, brute force checks 2ᵐ sub-collections — infeasible for large inputs.
Greedy provides an efficient approximation solution
Always choose the set covering the most uncovered elements. Guaranteed to be within a factor of H(k) ≈ ln(k) of optimal — fast, simple, and provably near-optimal.

Educational Overview NP-hardness, approximation ratio, and worked examples
›

Set Cover is an NP-hard optimization problem: given a universe U and a collection S of subsets, find the minimum number of subsets whose union equals U. The greedy algorithm provides a tractable approximation.

NP-Hardness
No polynomial-time algorithm is known that finds the optimal solution (unless P=NP). Set Cover is reducible from Vertex Cover and 3-SAT, placing it firmly in NP-Complete. This motivates the need for approximation algorithms.
Greedy Approximation Ratio
The greedy algorithm achieves: |C_greedy| ≤ H(max|Sᵢ|) · OPT
where H(k) = 1 + 1/2 + … + 1/k ≈ ln(k) + 0.577 (harmonic number).

Proof sketch: When the i-th element is covered, its "cost" is at most 1/|S*∩R| ≤ 1/(|U|−i+1). Summing over all |U| elements gives H(|U|) · OPT.
Inapproximability
Dinur & Steurer (2014) proved that unless P=NP, no polynomial-time algorithm can achieve an approximation ratio better than (1−ε) · ln|U| for any ε > 0. The greedy algorithm is therefore essentially optimal among efficient algorithms.
Worked Example — Classic 12-Element
U = {1…12}, S = { {1,2,3,4,5}, {1,4,7,10}, {2,5,8,11}, {3,6,9,12}, {4,5,6,7,8}, {9,10,11,12} }

Round 1: S1={1,2,3,4,5} covers 5 new → choose S1. R = {6…12}.
Round 2: S6={9,10,11,12} covers 4 new → choose S6. R = {6,7,8}.
Round 3: S5={4,5,6,7,8} covers 3 new (6,7,8) → choose S5. R = ∅.

Greedy: 3 sets. OPT ≥ ⌈12/5⌉ = 3. Ratio = 1 (optimal here).
H(5) = 1+0.5+0.333+0.25+0.2 = 2.283. Bound: 3 ≤ 2.283 · 3 ≈ 6.85. ✓
